# Аналіз платформ: Android, iOS, Web

## Крок 2. Загальна характеристика платформи

| Платформа | Тип | Основне призначення | Типи пристроїв |
|-----------|-----|-------------------|----------------|
| Android   | Мобільна | Споживчі пристрої, смартфони, планшети | Смартфони, планшети |
| iOS       | Мобільна | Споживчі пристрої, смартфони | Смартфони, планшети |
| Web       | Web     | Універсальна платформа для браузера | ПК, смартфони, планшети |

---

## Крок 3. Архітектура та середовище виконання

| Платформа | ОС | Ядро / архітектура | Runtime |
|-----------|----|------------------|--------|
| Android   | Android OS | Linux kernel, ARM/x86 | ART runtime |
| iOS       | iOS        | Darwin kernel, ARM   | Native runtime |
| Web       | Будь-яка ОС | Немає власного ядра | JS engine (браузер) |

---

## Крок 5. Доступ до апаратних ресурсів

| Ресурс          | Android | iOS | Web (браузер) |
|-----------------|--------|-----|---------------|
| Камера           | ✔ | ✔ | ✔ (через API) |
| Сенсори          | ✔ | ✔ | Обмежено      |
| Геолокація       | ✔ | ✔ | ✔ (через браузер) |
| Файлова система  | ✔ | ✔ | Обмежено (sandbox) |
| Мережа           | ✔ | ✔ | ✔            |
| Фонові процеси   | ✔ | ✔ | ❌ (обмежено) |

> **Примітка:** Web має обмеження на доступ до апаратних ресурсів через безпеку браузера.

---

## Крок 6. Модель безпеки

| Платформа | Модель дозволів | Sandboxing | Контроль доступу | Захист даних |
|-----------|----------------|------------|-----------------|--------------|
| Android   | Runtime permissions | ✔ | ✔ | ✔ |
| iOS       | Runtime permissions | ✔ | ✔ | ✔ |
| Web       | Запити користувача | ✔ (браузер) | ✔ | ✔ |

---

## Крок 7. UX та взаємодія з користувачем

- **Android:** Material Design, кнопки, жести, адаптивний інтерфейс  
- **iOS:** Human Interface Guidelines, жести, простий дизайн  
- **Web:** Responsive Design, адаптація під екран, обмеження браузера  

---

## Крок 8. Підтримка крос-платформної розробки

| Платформа | Фреймворки / технології | Легкість переносу | Runtime / Web |
|-----------|------------------------|-----------------|---------------|
| Android   | Kotlin Multiplatform, Flutter | Середня | ART runtime |
| iOS       | SwiftUI, Flutter, React Native | Середня | Native runtime |
| Web       | React, Angular, Vue | Висока | JS engine |

---

## Крок 9. Переваги та обмеження

| Платформа | Переваги | Обмеження |
|-----------|----------|-----------|
| Android   | Широка аудиторія, доступ до ресурсів, кастомізація | Фрагментація, різні версії OS |
| iOS       | Продуктивність, стабільність, безпека | Закрита екосистема, обмежений доступ до деяких ресурсів |
| Web       | Працює у будь-якому браузері, крос-платформність | Обмежений доступ до апаратних ресурсів, продуктивність залежить від браузера |

## Крок 11. Висновки

1. **Найбільш гнучка платформа:** Web (працює на будь-якій ОС та пристрої)  
2. **Для високої продуктивності:** iOS (оптимізація під апаратні ресурси)  
3. **Для швидкої розробки:** Web + фреймворки (React, Vue)  
4. **Для крос-платформних рішень:** Flutter, Kotlin Multiplatform  
5. **Сценарії використання:**  
   - Android → смартфони з широким охопленням  
   - iOS → продуктивні та безпечні мобільні додатки  
   - Web → універсальні рішення для різних пристроїв
